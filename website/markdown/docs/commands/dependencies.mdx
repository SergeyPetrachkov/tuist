# Dependencies (Chimera)

## How to get started

1. List requirements for third-party dependencies in `Dependencies.swift` manifest.
2. Install dependencies using `tuist dependencies [subcommand]` command.
3. Link dependencies with project's targets.
4. Regenerate project.

## `Dependencies.swift`

Requirements for third-party dependencies are defined in `Dependencies.swift` manifest. `Dependencies.swift` need to import the framework `ProjectDescription` which contains all the classes and enums that are available for you to describe your projects.

You can edit `Dependencies.swift` using `tuist edit` command.

Tuist expects `Dependencies.swift` under `Tuist/` directory:

```bash
AwesomeProject
	|- App
	|- AppTests
	|- Tuist
		|- Dependencies.swift
	|- Project.swift
```

The snippet below shows an example project manifest:

```swift
import ProjectDescription

let dependencies = Dependencies([
    .carthage(origin: .github(path: "Alamofire/Alamofire"), requirement: .exact("5.0.4"), platforms: [.macOS])
])
```

## Commands

The folder structure below shows how Tuist organize dependenies:

```bash
Tuist
	|- Dependencies.swift # Manifest
  |- Dependencies
    |- graph.json # comming soon
    |- Lockfiles
      |- Carthage.resolved
      |- Podfile.lock # comming soon
      |- Package.resolved # comming soon
    |- Carthage # stores content of `Carthage/Build` generated by `carthage`
      |- iOS
         |- Alamofire.framework
      |- tvOS
         |- Alamofire.framework
    |- Cocoapods # comming soon
      |- RxSwift
    |- SwftPackageManager # comming soon
      |- Moya
```

## Link dependencies

You can link third-party dependencies like local ones.

```swift
let target = Target(dependencies: [
  .framework("Tuist/Dependencies/Carthage/iOS/Alamofire.framework"),
])
```
